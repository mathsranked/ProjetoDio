O semeador

(Voltar)

Joseph Carl Robnett Licklider, mais conhecido como JCR Licklider, era um cientista da computação que é lembrado principalmente por ter defendido a criação da ARPANET, uma das primeiras Internet. Licklider organizou grande parte do financiamento e montou a equipe que eventualmente tornou a ARPANET uma realidade. Ele também é reconhecido como um dos primeiros teóricos em inteligência artificial e cibernética.

Techopedia explica Joseph Carl Robnett Licklider

O artigo de Licklider, de 1960, "Simbiose homem-computador", discutiu o futuro relacionamento de pessoas e computadores. No futuro, ele previa que os computadores trariam poder de processamento bruto e os humanos os guiariam de tarefa em tarefa. Em 1962, Licklider escreveu ainda mais sobre a necessidade do que era então chamado de rede galáctica. Depois de assumir a diretoria da Agência de Projetos de Pesquisa Avançada (APRA), Licklider colocou Larry Roberts no caminho da criação da Internet e também financiou o inovador Centro de Pesquisa em Ampliação (ARC) de Douglas Engelbart no Stanford Research Institute.

Browsers oque são?

(Voltar)

Um navegador de rede, navegador web, navegador da internet ou simplesmente navegador (em inglês: Web browser, browser), é um programa que habilita seus usuários a interagirem com documentos HTML hospedados em um servidor da rede.
Tim Berners-Lee, que foi um dos pioneiros no uso do hipertexto como forma de compartilhar informações, criou o primeiro navegador, chamado WorldWideWeb (www), em 1990.


Mais tarde, para não confundir-se com a própria rede, trocou de nome para Nexus. A web, entretanto, só explodiu realmente em popularidade com a introdução do NCSA 'Mosaic, que era um navegador gráfico (em oposição a navegadores de modo texto) rodando originalmente no Unix, mas que foi também portado para o Macintosh e Microsoft Windows logo depois. A versão 1.0 foi liberada em setembro de 1993. Marc Andreesen, o líder do projeto Mosaic na NCSA, demitiu-se para formar a companhia que seria conhecida mais tarde como Netscape Communications Corporation.

Proxy para que serve?

(Voltar)

Em redes de computadores , um servidor proxy é um aplicativo de servidor que atua como intermediário entre um cliente que solicita um recurso e o servidor que fornece esse recurso. [1] Em vez de se conectar diretamente a um servidor que pode atender a uma solicitação de um recurso, como um arquivo ou página da Web , o cliente direciona a solicitação ao servidor proxy, que avalia a solicitação e executa as transações de rede necessárias. Isso serve como um método para simplificar ou controlar a complexidade da solicitação ou fornecer benefícios adicionais, como balanceamento de carga , privacidade ou segurança. Os proxies foram criados para adicionar estrutura e encapsulamento a sistemas distribuídos . [2] Um servidor proxy, portanto, funciona em nome do cliente ao solicitar o serviço, mascarando potencialmente a verdadeira origem da solicitação para o servidor de recursos.

Tipos

Um servidor proxy pode residir no computador local do usuário ou em qualquer ponto entre o computador do usuário e os servidores de destino na Internet . Um servidor proxy que transmite solicitações e respostas não modificadas geralmente é chamado de gateway ou, às vezes, proxy de encapsulamento . Um proxy de encaminhamento é um proxy voltado para a Internet usado para recuperar dados de uma ampla variedade de fontes (na maioria dos casos, em qualquer lugar da Internet). Um proxy reverso geralmente é um proxy interno usado como front-end para controlar e proteger o acesso a um servidor em uma rede privada. Um proxy reverso geralmente também executa tarefas como balanceamento de carga , autenticação , descriptografia ecache .

Proxies abertos

Um proxy aberto é um servidor proxy de encaminhamento acessível por qualquer usuário da Internet. Em 2008, o especialista em segurança de rede Gordon Lyon estimou que "centenas de milhares" de proxies abertos são operados na Internet.
  • Proxy anônimo – Este servidor revela sua identidade como um servidor proxy, mas não revela o endereço IP de origem do cliente. Embora esse tipo de servidor possa ser descoberto facilmente, pode ser benéfico para alguns usuários, pois oculta o endereço IP de origem.
  • Proxy transparente – Este servidor não apenas se identifica como um servidor proxy, mas com o suporte de campos de cabeçalho HTTP , como X-Forwarded-For, o endereço IP de origem também pode ser recuperado. O principal benefício de usar esse tipo de servidor é a capacidade de armazenar em cache um site para uma recuperação mais rápida.

Proxies reversos

Um proxy reverso (ou substituto) é um servidor proxy que parece ser um servidor comum para os clientes. Proxies reversos encaminham solicitações para um ou mais servidores comuns que lidam com a solicitação. A resposta do servidor proxy é retornada como se viesse diretamente do servidor original, deixando o cliente sem conhecimento do servidor original. [5] Proxies reversos são instalados na vizinhança de um ou mais servidores web. Todo tráfego vindo da Internet e com destino a um dos servidores web da vizinhança passa pelo servidor proxy. O uso de reverse tem origem em sua contraparte forward proxyjá que o proxy reverso fica mais próximo do servidor da Web e atende apenas a um conjunto restrito de sites. Existem vários motivos para instalar servidores proxy reverso:
  • Criptografia/aceleração SSL: quando sites seguros são criados, a criptografia Secure Sockets Layer (SSL) muitas vezes não é feita pelo próprio servidor web, mas por um proxy reverso equipado com hardware de aceleração SSL. Além disso, um host pode fornecer um único "proxy SSL" para fornecer criptografia SSL para um número arbitrário de hosts, eliminando a necessidade de um certificado de servidor SSL separado para cada host, com a desvantagem de que todos os hosts por trás do proxy SSL precisam compartilhar um nome DNS comum ou endereço IP para conexões SSL. Esse problema pode ser parcialmente superado usando o recurso SubjectAltName dos certificados X.509 ou a extensão SNI do TLS .
  • Balanceamento de carga : o proxy reverso pode distribuir a carga para vários servidores web, cada servidor web atendendo a sua própria área de aplicação. Nesse caso, o proxy reverso pode precisar reescrever os URLs em cada página da Web (tradução de URLs conhecidos externamente para os locais internos).
  • Servir/armazenar em cache conteúdo estático: um proxy reverso pode descarregar os servidores da Web armazenando em cache conteúdo estático, como imagens e outros conteúdos gráficos estáticos.
  • Compactação : o servidor proxy pode otimizar e compactar o conteúdo para acelerar o tempo de carregamento.
  • Alimentação de colher: reduz o uso de recursos causado por clientes lentos nos servidores da Web, armazenando em cache o conteúdo que o servidor da Web enviou e "alimentando de colher" lentamente para o cliente. Isso beneficia especialmente as páginas geradas dinamicamente.
  • Segurança: o servidor proxy é uma camada adicional de defesa e pode proteger contra alguns ataques específicos do sistema operacional e do servidor da web. No entanto, ele não fornece nenhuma proteção contra ataques contra o próprio aplicativo ou serviço da Web, que geralmente é considerado a maior ameaça.
  • Publicação de extranet: um servidor proxy reverso voltado para a Internet pode ser usado para se comunicar com um servidor de firewall interno a uma organização, fornecendo acesso à extranet para algumas funções enquanto mantém os servidores atrás dos firewalls. Se usado dessa forma, medidas de segurança devem ser consideradas para proteger o restante de sua infraestrutura caso esse servidor seja comprometido, pois seu aplicativo web fica exposto a ataques da Internet.
